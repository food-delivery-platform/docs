# AI Chat Service — AI-помощник для клиентов

## Что делает

Умный чат в клиентском приложении — отвечает на вопросы без живого оператора.

- Отвечает на вопросы о статусе заказа (если передан контекст заказа)
- Подсказывает блюда с учётом аллергенов и диеты
- Рекомендует из меню
- Отвечает на FAQ: доставка, возвраты, оплата
- Ищет ответ в базе знаний платформы, а не «выдумывает из головы»
- Стримит ответ по частям, как в обычном мессенджере

## Кто пользуется

- **Клиентское приложение** — плавающий виджет чата в углу экрана

Другие приложения (ресторан, курьер, ops) этот сервис не используют в MVP.

## Связи с другими сервисами

| С кем | Зачем |
|---|---|
| **KB Ingestion Service** | База знаний наполняется FAQ, меню, политиками — чат ищет там |
| **Catalog Service** | Актуальное меню и описания ресторанов попадают в базу знаний |
| **Order Service** | Контекст текущего заказа для вопросов «где мой заказ?» |
| **User Service** | Предпочтения клиента (аллергены, диета) |

## Как работает (простыми словами)

1. Клиент пишет вопрос
2. Сервис ищет похожие фрагменты в базе знаний
3. Собирает промпт: системные правила + найденный контекст + история диалога
4. Отправляет в языковую модель (AWS Bedrock или OpenAI — настраивается)
5. Отдаёт ответ клиенту потоком

## Что хранит

Настройки модели (какая LLM, температура, сколько фрагментов искать). Сами знания — в базе, которую наполняет KB Ingestion Service. История чата в MVP не сохраняется (приватность).
