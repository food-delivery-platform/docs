# Delivery Service — Доставка и отслеживание

## Что делает

Всё после «еда готова»: курьер, дорога, карта, финальная доставка.

- Когда ресторан подтвердил заказ — ищет ближайшего свободного курьера и предлага ему задачу
- Принимает GPS-позицию курьера каждые ~10 секунд
- Ведёт этапы доставки: забрал → в пути → доставлено
- Показывает клиенту живое отслеживание на карте
- Передаёт обновления обратно в Order Service

## Кто пользуется

- **Приложение курьера** — список задач, принять/отклонить, кнопки этапов, отправка GPS
- **Клиентское приложение** — карта и статус «курьер едет»
- **Ops-дашборд** — карта всех курьеров в реальном времени

## Связи с другими сервисами

| С кем | Зачем |
|---|---|
| **Order Service** | Получает событие «заказ подтверждён»; отправляет обновления статуса доставки |
| **User Service** | Курьеры — пользователи с ролью COURIER |
| **Notification Service** | События вроде «курьер забрал заказ», «скоро будет у вас» |
| **Monitoring Service** | Контроль: не застряла ли доставка слишком долго |

## Этапы доставки (упрощённо)

```
Заказ готов → Курьер назначен → Забрал → В пути → Доставлено
```

## Что хранит

Назначения курьеров на заказы, текущие GPS-координаты, журнал событий по доставке.

Это единственный сервис с постоянным онлайн-соединением (WebSocket) — курьеры и клиенты должны видеть обновления без задержки.
